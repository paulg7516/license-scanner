License Scanner

A self-service web portal for managing SaaS license optimization. Scans platforms like PagerDuty, Atlassian, and GitLab to identify inactive users, surface cost savings, and monitor API integration health with a full audit trail.

Getting Started

Prerequisites

  • Python 3.10+
  • An Anthropic API key (for AI Insights)
  • Azure AD App Registration (for production SSO -optional for local dev)

Local Setup

# Clone the repo
git clone https://github.com/paulg7516/license-scanner.git
cd license-scanner

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your values (see Environment Variables below)

# Run
streamlit run app.py --server.port 8501 --server.headless true

Open http://localhost:8501 and log in with the access code set in your .env file.

User Guide

Navigation

The portal has four main tabs plus sign out:

Scan Overview AI Insights Audit Log Integrations

Scan Overview

License Optimization
Scan Overview
View inactive users, track savings, and manage license statuses across all connected platforms.

Headline Metrics

  • Realized savings (from deactivated users)
  • Pending action (unresolved inactive users)
  • Total identified savings (monthly and annual)

Running a Scan

  • Click Run Scan to start scanning all connected platforms
  • A progress bar shows real-time status (current platform, % complete)
  • Click Stop Scan to cancel a running scan

Managing Inactive Users

Each organization section shows inactive users grouped by platform. For each user you can see their name, email, last active date, and monthly cost.

  1. Use the status dropdown next to each user to change their status
  2. Or select multiple users with checkboxes and use bulk actions
StatusMeaning
PendingFlagged as inactive, no action taken yet
DeactivatedLicense reclaimed -counts toward realized savings
KeptIntentionally retained (business justification)

Filters: Filter by platform using the dropdown, or search by status text.

Verifying Tokens

  • Click Verify All to test all platform tokens with a lightweight API call

  • Credentials are read from the backend watchdog config (not stored in the UI)

Audit Log

Compliance
Audit Log
A complete activity history for compliance and accountability.

What's Tracked

  • Scan started / completed / cancelled
  • User status changes (pending, deactivated, kept)
  • Bulk status actions

Controls: Filter by system or view all events. Paginated at 15 entries per page. Each entry shows action, source system, details, user, and timestamp.

AI Insights

AI-Powered Analysis
Executive Summary
Intelligent analysis of your scan data to surface what matters most.

Generating a Summary

  1. Navigate to the AI Insights tab
  2. Click Generate Summary
  3. The AI analyzes your latest scan data and produces:
  • Headline -one-sentence overview
  • Key Findings -what the scan uncovered
  • Opportunities -where the biggest savings are, with dollar amounts
  • Recommended Actions -concrete next steps
  • Bottom Line -strategic recommendation

Caching: Summaries are cached and reused until new scan data is available. Click Generate Summary again to force a refresh.

Technical Reference

Architecture

license-scanner/
├── app.py              # Main Streamlit app -all UI logic
├── auth.py             # Azure AD OAuth2 via MSAL
├── config.py           # Environment config + SYSTEMS registry
├── storage.py          # Encrypted token storage (SQLite + Fernet)
├── ai_summary.py       # Claude API integration for executive summaries
├── requirements.txt    # Python dependencies
├── render.yaml         # Render.com deployment blueprint
├── .env.example        # Environment variable template
├── assets/             # Logo images
└── Data Files:
    ├── scan_results.json       # Latest scan output
    ├── scan_history.json       # Historical scan data
    ├── user_actions.json       # User status decisions
    ├── token_validation.json   # API token validation results
    ├── user_registry.json      # Known users
    ├── audit_log.json          # Activity log (last 500 entries)
    ├── ai_summary.json         # Cached AI summary
    └── watchdog.db             # SQLite database

Data Flow

1
An external scanner connects to platform APIs and writes scan_results.json
2
The UI reads and renders scan results
3
The Integrations page makes lightweight API calls to validate token health (reads credentials from the backend config)
4
User actions (status changes) are saved to JSON files and SQLite
5
The audit log captures all state changes with who, what, and when
6
AI summaries are generated on demand from scan data via the Claude API

Authentication

Access Code: A shared access code (set via ACCESS_CODE env var) provides simple authentication for internal users.

Microsoft SSO: Azure AD SSO via MSAL authorization code flow is available as an alternative login method. User groups from Azure AD control which systems a user can manage.

Security

  • No credentials in the UI -API tokens live in the backend watchdog config; the UI only reads them for validation and never displays them
  • Group-based access control -Azure AD groups restrict system management
  • Audit trail -every action is logged with user identity and timestamp

Deployment

Render.com

The repo includes a render.yaml blueprint for one-click deployment.

  1. Push to GitHub
  2. On Render: New > Web Service > connect repo
  3. Render auto-detects the blueprint
  4. Set environment variables in the dashboard: ENCRYPTION_KEY, ANTHROPIC_API_KEY, and Azure AD vars for production
  5. Set REDIRECT_URI to your Render URL

Start command:

streamlit run app.py --server.port $PORT --server.headless true --server.address 0.0.0.0

Note: Render's free tier has an ephemeral filesystem -JSON data files and SQLite reset on redeploy. Use Render's Disk add-on for persistence.

Adding New Platforms

To add a new platform to scan and track:

  1. Register the system in config.py
  2. Update the scanner to connect to the new platform's API
  3. Deploy -the UI automatically picks up new systems from the config

            {"key": "base_url", "label": "Instance URL", "type": "text"},
        ],
    },
}
